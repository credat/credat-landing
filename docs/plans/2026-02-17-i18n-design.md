# i18n Design — Credat Landing Page

**Date:** 2026-02-17
**Status:** Approved
**Library:** next-intl

## Languages

EN (default), FR, DE, ES — covers ~70% of EU population. Expandable later.

## URL Strategy

Subpath routing with `localePrefix: 'as-needed'`:
- `/` — English (default, no prefix)
- `/fr` — French
- `/de` — German
- `/es` — Spanish

## Architecture

```
src/
├── i18n/
│   ├── routing.ts          # defineRouting (locales, default, prefix)
│   ├── navigation.ts       # createNavigation (Link, useRouter, etc.)
│   └── request.ts          # getRequestConfig (message loading)
├── messages/
│   ├── en.json
│   ├── fr.json
│   ├── de.json
│   └── es.json
├── middleware.ts            # Locale detection + routing
├── app/
│   ├── [locale]/
│   │   ├── layout.tsx      # Locale-aware metadata + JSON-LD
│   │   └── page.tsx        # Homepage
│   ├── robots.ts
│   ├── sitemap.ts          # Multi-locale URLs
│   ├── icon.png
│   └── globals.css
└── components/
    ├── navbar.tsx           # + locale switcher (EN/FR/DE/ES dropdown)
    └── sections/...         # useTranslations() for all text
```

## Translation Files

- Namespaced by component/section (flat structure)
- ICU syntax for dynamic values: `{year}`, `{count}`
- Arrays use object keys (not indices) for maintainability
- Code snippets NOT in translation files — hardcoded in English
- Technical terms (eIDAS, SD-JWT, mDoc, etc.) stay untranslated

### Example structure:

```json
{
  "Metadata": { "title": "...", "description": "..." },
  "Navbar": { "useCases": "...", "integrations": "..." },
  "Hero": { "badge": "...", "title": "...", "ctaPrimary": "..." },
  "UseCases": { "title": "...", "items": { "banking": { "title": "...", "description": "..." } } },
  "Footer": { "copyright": "© {year} Credat. All rights reserved." }
}
```

## Locale Switcher

- Language code dropdown (EN/FR/DE/ES) in navbar
- Uses next-intl's `useRouter` to switch locale
- Positioned at far right of nav

## Content Decisions

| Content | Decision |
|---------|----------|
| Testimonials | Fully translated |
| Code snippets | Stay in English |
| Technical terms | Stay untranslated |
| Code comments | Stay in English |

## SEO

- `generateMetadata` in `[locale]/layout.tsx` with locale-aware title, description, OG
- `openGraph.locale` per language (en_US, fr_FR, de_DE, es_ES)
- `alternates.languages` for hreflang tags
- Sitemap generates URLs for all locales with alternates
- JSON-LD with locale-aware name/description and `inLanguage`

## Translation Method

AI-generated for all 4 languages, professional review later.

## What Stays the Same

- All GSAP animations (re-render handles locale change naturally)
- Component structure
- Styling / Tailwind classes
- Code snippets content
- External links
- Loading screen
